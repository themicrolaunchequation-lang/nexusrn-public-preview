Accessibility

Accessibility Statement

NexusRN is being prepared toward WCAG-oriented accessibility. The public preview should be tested with keyboard-only navigation, screen readers, reduced motion, high contrast, and mobile viewports before institutional use.

Current commitments

  • Readable color contrast in light/dark surfaces.
  • Visible focus states for interactive controls.
  • Reduced-motion CSS support where possible.
  • Clear recovery paths from demo and route screens.

Known preview limitations

  • A third-party WCAG audit has not yet been completed.
  • Complex NGN item interactions still need assistive-technology testing.
  • Iframe-based route behavior needs additional keyboard/back-button validation.

Reference standard

WCAG 2.2 is organized around making web content more perceivable, operable, understandable, and robust. See the official W3C guideline: Web Content Accessibility Guidelines 2.2.
